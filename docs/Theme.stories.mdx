# Intro to the Theme

Our theme is where we define global styles. This includes palette, spacings, typography and init. The files can be found at this directory:

```
btf-frontend
  | src
    | theme
      | index.js
      | init.styles.js
      | palette.styles.js
      | shadows.styles.js
      | spacings.styles.js
      | typography.styles.js
```

We're using [**styled-components**](https://styled-components.com/) a CSS in JS library to style this app. This is why our style files are `.js` not `.css`. However, unlike other CSS in JS libraries, styled-components uses the same syntax as regular css as opposed to generic JS object syntax.

_**You shouldn't need to edit these files!**_ - We've included this page so you can understand what's going on under the hood. To understand how to make use of theme in your code, take a look at the specific sections in the sidebar.

<br />

## What's in the files?

```js
// index.js

import React from 'react';
import Init from './init.styles';
import Palette from './palette.styles';
import Shadows from './shadows.styles';
import Spacings from './spacings.styles';
import Typography from './typography.styles';

const Theme = () => {
  return (
    <>
      <Init />
      <Palette />
      <Typography />
      <Spacings />
      <Shadows />
    </>
  );
};

export default Theme;
```

**index.js** - Nothing special happening here, we import all of the other theme files and export them as a single React component.

<br />

```js
// init.styles.js

import { createGlobalStyle } from 'styled-components';

const Init = createGlobalStyle`
  * {
    margin: 0;
    padding: 0;
    -webkit-font-smoothing: antialiased;
    box-sizing: border-box;
  }

  button,
  button:active,
  button:focus {
    background: none;
    outline: none;
    border: none;
  }

  a,
  a:active,
  a:focus {
    text-decoration: none;
    color: inherit;
    outline: none; /* Works in Firefox, Chrome, IE8 and above */
  }

  /* Set base as ten so rem units have a standardised base point */
  html {
    font-size: 10px;
  }

  ol,
  ul {
    list-style: none;
  }

  table {
    border-collapse: collapse;
    border-spacing: 0;
  }
`;

export default Init;
```

**init.styles.js** - This is the structure the rest of the files in the directory will use. We import `createGlobalStyle` from styled-components and pass it a template string of CSS. This file provides some default overides to allow for more consistent styling.

<br />

```js
// palette.styles.js

import { createGlobalStyle } from 'styled-components';

const Palette = createGlobalStyle`
  :root {
    --lightpink: #F28383;
    --darkpink: #BF3459;
    --mattepink: #A65369;
    --logopink: #F20544;
    --lightgrey: #F2F2F2;
    --darkgrey: #737373;
    --darkburgundy: #440113;
    --orange: #F29C50;
    --lightblue: #05AFF2;
    --midblue: #006CE6;
    --darkblue: #272A44;
  }
`;

export default Palette;
```

**palette.styles.js** - We're defining our colors, and all other theme variables, using [**CSS custom properties**](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties). These can be accessed anywhere we use styled-components the same way you would in CSS:

`background-color: var(--lightpink):`

As we've defined light pink as `#F28383` the above code will set he background color to that hex value.

<br />

```js
// shadows.styles.js

import { createGlobalStyle } from 'styled-components';

const Shadows = createGlobalStyle`
  :root {
    --card-shadow: 0 1rem 1rem rgba(0, 0, 0, 0.1);
    --modal-shadow: 0 0 6rem rgba(0, 0, 0, 0.5);
  }
`;

export default Shadows;
```

**shadows.styles.js** - Similar to our palette file, shadows uses predefined `box-shadow` values so that all of our components are created with consistent shadow appearances.

<br />

```js
// spacings.styles.js

import { createGlobalStyle } from 'styled-components';

const Spacings = createGlobalStyle`
  :root {
    --smallest-space: 1rem;
    --small-space: 2rem;
    --medium-space: 4rem;
    --large-space: 8rem;
    --largest-space: 16rem;
  }
`;

export default Spacings;
```

**spacings.styles.js** - Spacings uses predefined rem values so all our components are created with consistent sizings.
